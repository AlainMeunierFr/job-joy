# Sprints — méthodologie

**Un dossier par Sprint.** Contenu défini par le projet.

## Structure type

- **`00 - Sprint goal et contexte.md`** : objectif du sprint et contexte (optionnel).
- **`US-[n°Sprint].[n°US] - [Titre US].md`** : une User Story = un fichier (ex. `US-1.1 - Titre.md`).

## Pilotage des US

- Le pilotage des US actives se fait par le **Lead Dev en mémoire de session (RAM)**.
- Ne pas créer de fichier d'état dédié pour piloter les US.
- `GO NEXT` doit cibler explicitement la bonne US/lane en cas de travail parallèle.
